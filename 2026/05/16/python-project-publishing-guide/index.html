四、国内网络环境的三个坑

4.1 git push 被 GFW 阻断（HTTPS 写入）

现象：git clone 和 git fetch 正常，但 git push 超时。

原因：GFW 的 DPI 识别 git-receive-pack 协议后静默丢包，只拦截写入不拦截读取。

解决：切换到 SSH 协议。

前提：本地需配置 SSH key 并添加到 GitHub。

4.2 SSH 也偶尔不通 —— GitHub API 兜底

GFW 封锁是间歇性的。当 SSH 也不通时，使用 GitHub REST API 推送提交：

API 走标准的 HTTPS 443 端口，不会被 DPI 识别。

4.3 PyPI 上传超时

twine upload 在 GFW 下上传大型包可能卡在 0%。两个方案：

  • 方案 A：用户手动上传（60KB 的 wheel 文件通常能过）
  • 方案 B：打 tag 让 GitHub Actions 自动发布（推荐，Actions 跑在境外服务器）

五、完整发布 checklist

首次配置（一次性）

  • GitHub 仓库创建完成
  • PyPI 项目名 pip install <名字> 测试通过
  • GitHub Secrets：PYPI_TOKEN + CLOUDFLARE_API_TOKEN
  • Cloudflare Pages 项目创建
  • SSH key 配置（GFW 环境下必做）
  • site/ 加入 .gitignore

六、总结

从零到完整发布一个 Python 开源项目，核心工作流就三步：

关键决策：

  • 文档站：Cloudflare Pages（免费 + 国内可访），不要用 GitHub Pages
  • 认证：PyPI 用 API Token，Cloudflare 用 API Token，GitHub 用 SSH
  • 自动化：publish.yml（tag 触发）+ docs.yml（push 触发）

在此架构下，开发者的日常就是改代码 + 打 tag，剩下的全部自动完成。
